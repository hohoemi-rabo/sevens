# 14. 音声・効果音

**フェーズ**: 4　**依存**: 05　**対象**: `src/components/audio/`, `public/audio/`

## 概要
「ワイワイ感」を生む中心機能として、音声読み上げと効果音を実装する。各端末で個別に音量調整・ミュートできる。

## 関連要件
- `REQUIREMENTS.md` 3.5（音声・効果音）、6.4

## 仕様メモ（実装で確定）
- **読み上げ = Web Speech API（音声合成）**。録音ファイル不要・LANオフライン動作・全52枚に動的対応（`src/lib/audio/speech.ts`）。要件の「事前録音」からは外れるが、教室運用での実用性・保守性を優先。
- **効果音 = mp3（`public/audio/`）**。固定ファイル名で事前ロードし、**未配置でも throw せず無音フォールバック**（`src/lib/audio/sfx.ts`）。後から mp3 を置けば鳴る。期待名は `public/audio/README.md` 参照。
- **イベント検出はサーバー状態の差分**で行う（`game:state` のみ購読、出札/パスに専用イベントは無い）。前後 GameState を純関数 `diffGameState`（`src/lib/audio/events.ts`）で差分し、出札札の特定・脱落の非誤検出・上がり=play+finish・パス・配札・終局を導出。
- **ベースラインガード**: セッション最初の状態は基準扱い（配札のみ判定）。再接続でゲーム途中に復帰しても過去イベントを一斉再生しない。
- 音量・ミュートは端末ごと（`audioStore`）。デフォルト中音量（0.5）。`sessionStorage` に永続（`audio-settings-storage.ts`、CLAUDE.md の唯一の永続化例外に整合）。
- 自動再生ポリシー対策: 初回ユーザー操作で音声を解禁（`src/lib/audio/unlock.ts`）。
- ターン通知音は #15（お助けモード）の範囲。結果画面の演出作り込みは #17。
