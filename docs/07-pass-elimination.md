# 07. パス回数・脱落管理

**フェーズ**: 2　**依存**: 03　**対象**: `src/lib/sevens/pass.ts`

## 概要
パス回数の上限管理と、上限超過時の脱落処理（手札を全て場に出す）を実装する。

## 関連要件
- `REQUIREMENTS.md` 3.2「パスのルール（重要）」、3.6

## 仕様メモ
- **パス可能回数はホストが部屋作成時に設定（1〜5回）**。ゲーム設定値として保持
- 残りパス回数は各プレイヤーごとに管理し、UIに常時表示（表示は 16/15 側）
- 設定回数を**超えてパス**した場合、そのプレイヤーは**手札を全て場に出して脱落**
  - 「手札を全て場に出す」処理の仕様を明確化（出せる札だけか、強制的に全部か）。標準解釈は**全部場に出して脱落＝以後の手番から除外**
- 脱落は順位（08）とは別枠で記録する

## 実装メモ
- ホスト決定により**脱落者の手札は本来ルール準拠で実際に盤面へ配置**する。これに対応するため
  盤面モデルを `{low,high}` 連続範囲 → **配置済みランクの配列 `Record<Suit, Rank[]>`**（隙間表現可・JSON安全）へ拡張（`board.ts`）。`placeForced()` で一括放出。`runAround7()` で7を含む連続ブロックの端だけが出せる（隙間の先の孤立札は埋まるまで出せない＝本来ルール）。
- パスルールは `pass.ts`（純粋TS）: `isValidMaxPass`(1〜5) / `willEliminateOnPass`(残0で次パス) / `isWastefulPass`(15のお助け警告用)。型は state.ts から type-only import で循環回避。
- `state.ts`: `Player.eliminatedOrder` 追加（順位 rank とは別枠、08で利用）。`pass()` が超過時に放出＋脱落へ遷移。手番除外は既存 `advanceTurn` がそのまま処理。
- 完走（デッドロックせず終了）と脱落の振る舞いは state/cpu/pass/board の各テストで担保。

## 完了条件
- パス上限の設定が反映され、超過で正しく脱落する
- 脱落者が以降の手番から除外される
