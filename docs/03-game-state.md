# 03. ゲーム状態・ターン進行

**フェーズ**: 1　**依存**: 01, 02　**対象**: `src/lib/sevens/state.ts`

## 概要
1ゲーム分の状態（プレイヤー・手札・場・手番・パス残数・順位/脱落）を保持し、「カードを出す」「パスする」といった操作で状態遷移する純粋ロジックを実装する。状態は通信層・Zustandから利用される単一の信頼できる情報源（SSOT）。

## 関連要件
- `REQUIREMENTS.md` 3.6（対局フロー）、7.1、5.3（メモリ上のみで管理）

## 仕様メモ
- プレイヤー：人間＋CPUで4人固定。各プレイヤーに id / 名前 / 席番号 / 手札 / 残パス回数 / 状態（プレイ中・上がり・脱落）
- 手番は席順で進行。上がり/脱落者はスキップ
- 操作：`playCard`（07の脱落・08の順位判定と連携）、`pass`
- 状態は**イミュータブルに更新**（新しい状態を返す）すると、テスト・再接続復元・通信同期がしやすい
