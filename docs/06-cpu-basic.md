# 06. CPU1人と対戦できる状態

**フェーズ**: 1　**依存**: 03, 05　**対象**: `src/lib/sevens/cpu/`

## 概要
最低限のCPU思考（出せる札があれば出す、なければパス）を実装し、人間1人＋CPUで対局が最後まで回る状態にする。弱・中・強の作り分けは 12 で行う（ここではまず「弱」相当）。

## 関連要件
- `REQUIREMENTS.md` 3.4（CPUプレイヤー）、開発ロードマップ フェーズ1（CPU1人と打てる状態）

## 仕様メモ
- まずは「弱」相当：出せる札があればほぼ即出し、なければパス
- 思考時間の演出（1〜2秒）はUI側で待機を入れる（`REQUIREMENTS.md` 3.4）
- CPUのインターフェースを `(state, playerId) => Action`（出す/パス）の形に統一し、12で弱/中/強を差し替えられるようにする

- `src/lib/sevens/cpu/`: `types.ts`（`Action` / `CpuStrategy`）, `weak.ts`（`decideWeak`）, `index.ts`。中・強は12で同インターフェースで差し替え。
- 自動進行は `GameTable.tsx` の `useEffect`＋`setTimeout`（演出1.2秒）。人間=席0、席1〜3がCPU（弱）。
- 完走（デッドロックしないこと）は全員CPUの単体テスト `cpu/weak.test.ts` で検証。脱落処理は未実装のため07で対応。
