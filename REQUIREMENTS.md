# 7並べ（セブンズ）ゲーム 要件定義書

## 1. プロジェクト概要

### 1.1 目的
ほほ笑みラボ（パソコン・スマホ教室）の生徒さん向けに、教室内のWi-Fi環境で気軽に楽しめる4人対戦の7並べを提供する。麻雀ゲームで構築した通信層・部屋管理・CPU・お助けモード・音声の資産を最大限流用し、**初心者でもワイワイ楽しめる卓上ゲーム体験**を最優先とする。

### 1.2 コンセプト
- **「ワイワイ楽しむ」を最優先**：卓を囲むコミュニケーションの場
- **初心者ファースト**：ルールを知らなくても遊べる「お助けモード」
- **音声・効果音重視**：視覚＋聴覚で状況が伝わる
- **シニアファースト**：大きな文字、迷わないUI、誤操作リカバリ
- **PC画面に最適化**：実際のプレイはPC（横長画面）で行う

### 1.3 ターゲットユーザー
- ほほ笑みラボの生徒さん（シニア、ほぼ全員がトランプ初心者〜カジュアル層）
- 講師（先生）が進行役兼ホストとして対局にも参加

### 1.4 麻雀ゲームからの流用方針
本プロジェクトは既存の麻雀ゲームと同じ技術基盤を共有する。以下は流用・踏襲する：
- 通信層（Socket.io、LocalAdapter）
- 部屋作成・合言葉入室・QRコード・席割り
- CPU補完の仕組み（弱・中・強の3段階）
- お助けモードのオンオフ切替の思想
- 音声・効果音の再生基盤
- シニア向けUI原則（大きな文字・高コントラスト・誤操作リカバリ）
- 記録・保存なし（その場限り）の方針

## 2. 動作環境

### 2.1 端末構成
- **ホスト端末**：ノートPC1台（先生が操作。プレイヤーとしても参加）
- **参加端末**：PCまたはタブレット（生徒さん用に最大3台）
- **メインターゲットはPC**（横長画面でのUI/UXを最優先で設計）
- **ネットワーク**：教室内のローカルWi-Fi（LAN）
- **クラウド・サーバー不要**：ホストPC1台で完結

### 2.2 接続方法
1. ホストPCで起動 → 画面にQRコードとURL（例：`http://192.168.x.x:3000`）を表示
2. 参加端末はQRコードまたはURL＋4桁合言葉で入室
3. 名前入力 → 席選択 → 対局開始

### 2.3 画面方向
- **PC横長画面（ランドスケープ）を基本**とし、最優先で最適化
- タブレットでも崩れないようレスポンシブ対応

## 3. 機能要件

### 3.1 基本機能
- 部屋作成・参加（4桁合言葉、QRコード表示）
- 席割り（4人席の自動割当、または手動選択）
- カードの配布・場への配置・パス・上がり判定
- **CPU補完**（参加者が4人未満の場合、自動でCPUを追加）
- 4人固定（トランプ52枚を4人で分配）

### 3.2 ゲームルール

#### 基本ルール
| 項目 | 仕様 |
|------|------|
| 使用カード | トランプ **52枚**（ジョーカーなし／標準ルール） |
| プレイ人数 | 4人（人間＋CPUで4人を埋める） |
| 配札 | 52枚を4人に均等配分（各13枚） |
| 開始 | ♦7を持つ人からスタート（標準ルール）。または各スートの7から場に置いて開始 |
| 進行 | 場に出ているカードの隣（連続する数字）に手札を1枚ずつ出す |
| スートの広がり | 7を起点に、8→9→…→K、6→5→…→A の両方向へ伸ばす |

#### パスのルール（重要）
- **パス可能回数はホストが設定**（1〜5回の範囲で部屋作成時に指定）
- 設定回数を超えてパスした場合、**手札を全て場に出して脱落**
- 残りのパス回数は各プレイヤーの画面に常時表示

#### 勝敗・順位（フル仕様）
- 手札を最も早く出し切った順に **1位・2位・3位・4位** を決定
- **脱落者**（パス超過で手札を場に出した人）も明示的に表示
- 全員が上がるか脱落するまで対局を継続
- 最終結果画面で順位と脱落を一覧表示

### 3.3 お助けモード（オンオフ切替）
**画面上のトグルスイッチで切替可能。デフォルトはオン。**

#### お助けモードON時
- **出せる札をハイライト表示**：手札のうち今出せるカードを強調（一目で分かる）
- **パス警告**：出せる札があるのにパスしようとしたら「出せる札がありますが、パスしますか？」と確認ダイアログ
- 残りパス回数の強調表示
- 自分の番が来たことの明示（ターン通知）

#### お助けモードOFF時
- 自分の番の通知のみ（最低限のサポート）

### 3.4 CPUプレイヤー
**弱・中・強の3段階**から選択可能（部屋作成時にホストが設定）

| 強さ | 特徴 |
|------|------|
| 弱 | 出せる札があればほぼ即出し。戦略性なし。生徒さんが勝ちやすい |
| 中 | 自分の手札の連続性を見て、止めると有利な札は少し溜める |
| 強 | 相手の動きを読み、キーカード（端に近い札）を戦略的に止める |

- CPUの思考時間：1〜2秒（待たせすぎない演出）
- 各CPUに名前とアバターを付与（添付画像のように「りつこ」「ハジメ」「ミミ」等のキャラクター）

### 3.5 音声・効果音
**「ワイワイ感」を出す中心機能。麻雀と同じ方針。**

#### 音声読み上げ
- カードを出したときの読み上げ（例：「ダイヤの8！」）
- パス宣言
- 上がり（「あがり！」）
- 脱落（「残念！」）

#### 効果音
- カードを出す音
- パス時のSE
- 上がり時の**拍手音**
- 配札時のシャッフル音
- 脱落時のSE

#### 音量設定
- 各端末で個別に音量調節・ミュート可能
- デフォルトは中音量

### 3.6 対局フロー
1. ホストが「部屋を作る」→ 合言葉とQRコード表示
2. ホストがパス回数・CPU強さを設定
3. 参加者がQRコードまたはURL＋合言葉で入室 → 名前入力 → 席選択
4. ホストが「CPUで埋める」ボタンを押す（人数不足時）
5. 「対局開始」→ 配札アニメーション＋シャッフル音
6. ♦7（または各7）から場がスタート
7. 順番にカードを出す or パス（残り回数管理）
8. 出し切った人から順位確定、パス超過者は脱落
9. 全員終了 → 結果画面（順位＋脱落、拍手音）
10. 「もう一回」or「部屋を解散」を選択

### 3.7 シニア向けUI配慮
- **大きなカード・大きなボタン**（最小タップ領域 60×60px以上）
- **高コントラスト配色**（スートの赤・黒をはっきり、数字を大きく）
- **「待って」ボタン**：自分のターンを一時停止できる
- **出す前の確認**：「このカードを出しますか？」（オンオフ可、デフォルトオン）
- **明瞭なナビゲーション**：今は誰の番か、何をすればいいかが一目で分かる
- **読みやすい日本語**：用語に補足（例：「パス（出せないとき）」）

## 4. 画面レイアウト（PC横長最適化案）

添付画像（スマホ縦画面）の構成を踏襲しつつ、**PC横長画面向けに再構成**する。

### 4.1 全体構成
```
┌────────────────────────────────────────────────────────┐
│  [メニュー]                                  [音量][お助けON/OFF] │
├────────────────────────────────────────────────────────┤
│                                                        │
│   ┌──────┐      ┌──────┐      ┌──────┐               │
│   │りつこ │      │ハジメ │      │ ミミ  │  ← 対戦相手3人  │
│   │残り12 │      │残り11 │      │残り11 │   （上部に横並び）│
│   └──────┘      └──────┘      └──────┘               │
│                                                        │
│        ┌──────────────────────────────┐               │
│        │  ♠ 　A 2 3 4 5 6 [7] 8 9 …K │               │
│        │  ♥ 　　　　　　　 [7] 8 …     │ ← 場（中央に大きく）│
│        │  ♦ 　　　　　　　 [7] 8 …     │   4スート横並び   │
│        │  ♣ 　　　　　　　 [7] …       │               │
│        └──────────────────────────────┘               │
│                                                        │
│   ┌──────────────────────────────────────┐           │
│   │  手札（自分）大きく横並び              │ ← 下部       │
│   │  2 3 J 5 4 5 10 K 2 3 5 [6]           │           │
│   └──────────────────────────────────────┘           │
│                                                        │
│   [パス（残りn回）]      YOU      [出す ↑]              │
└────────────────────────────────────────────────────────┘
```

### 4.2 レイアウトのポイント
- **対戦相手3人**：画面上部に横並び。アバター・名前・残り枚数・残りパス回数を表示
- **場（4スート）**：画面中央に大きく配置。横長画面を活かし、A〜Kが横一列に並ぶゆとりあるレイアウト。7を中央基準にして左右に伸びる様子が直感的に分かる
- **自分の手札**：画面下部に大きく横並び。出せるカードはハイライト（お助けON時）
- **操作ボタン**：左下に「パス（残り回数表示）」、右下に「出す」。中央に自分のアバター
- **常時情報**：右上に音量・お助けトグル、左上にメニュー
- **PC最適化**：マウスクリックでカード選択、ホバーでハイライト。横幅を活かしてカードを大きく、間隔にゆとりを持たせる

### 4.3 カードデザイン
- **添付画像のようなカジュアルで親しみやすい雰囲気**
- 数字・スートを大きく明瞭に（シニアの視認性優先）
- 赤（♥♦）・黒（♠♣）のコントラストをはっきり
- 出せるカードのハイライト、出せないカードのグレーアウトを明確に区別
- SVGで自作（麻雀牌と同じ手法。ライセンス問題回避・調整自由・差し替え容易）

## 5. 非機能要件

### 5.1 パフォーマンス
- 同一LAN内で4端末同時接続可能
- 通信遅延：100ms以内
- 起動：ホストPCでダブルクリック起動（将来Tauri/Electronで実現）

### 5.2 信頼性
- 参加端末が切断した場合、5秒以内に再接続可能
- 再接続後、対局状態を復元
- ホストPCが落ちた場合は対局終了（1ゲーム制なので影響軽微）

### 5.3 データ管理
- **記録・保存なし**：対局結果はその場限り
- **データベース不要**
- 状態はメモリ上のみで管理
- 個人情報は名前のみ（永続化なし）

### 5.4 セキュリティ
- 教室内LANのみで動作（インターネット不要）
- 4桁合言葉による入室制限
- 認証機能なし（教室内クローズド環境前提）

## 6. 技術スタック

### 6.1 フロントエンド
- **Next.js（App Router）+ TypeScript**
- **Tailwind CSS**（シニア向け大きな文字・高コントラスト）
- **Zustand**（状態管理）

### 6.2 通信
- **Socket.io**（クライアント・サーバー両方）
- カスタムサーバー（`server.ts`）でNext.jsとSocket.ioを同居
- 麻雀ゲームのLocalAdapterを流用

### 6.3 ゲームロジック
- **純粋なTypeScriptモジュール**（通信層から分離）
- 7並べのルール処理（配札・出せる札判定・パス管理・順位判定）

### 6.4 音声
- **HTMLAudioElement** または **Web Audio API**
- 音声ファイル（mp3）は事前ロード
- 事前録音した音声ファイルを使用

### 6.5 グラフィック素材
- **トランプカードはSVGで自作**（Claude Codeで生成）
- カジュアルで親しみやすいデザイン、シニア向けに数字・スートを大きく
- `public/cards/` 配下に配置

**命名規則（例）**
- スペード：`s1.svg`（A）〜 `s13.svg`（K）
- ハート：`h1.svg` 〜 `h13.svg`
- ダイヤ：`d1.svg` 〜 `d13.svg`
- クラブ：`c1.svg` 〜 `c13.svg`
- 裏面：`back.svg`

### 6.6 配布・起動
- 開発時：`npm run dev` で起動
- 教室運用時：将来的にTauri / Electronでワンクリック起動アプリ化を検討

## 10. 成功指標

定量指標は設けないが、以下を達成できれば成功とする：
- 教室の生徒さんが**ルールを知らなくても**最後まで遊び切れる
- 拍手音や読み上げで**自然と笑いや会話が生まれる**
- 先生が**準備に手間取らず**起動できる
- CPUを混ぜても**プレイの流れが止まらない**
- **PC画面で見やすく**、マウス操作で迷わない
